簡單三步，實現上雲

專為開發者設計的極簡整合流程

01. 快速接入 使用我們的 SDK (Go/C++/Python/JS) 或是透過 REST API，讓你的機器人連接到 KairoIO。
02. 定義 Schema 在網頁後台定義你的訊息與動作格式。無需修改機器人端代碼，Dashboard 自動適配。
03. 啟動管理 打開手機 App 或瀏覽器 Dashboard，隨時隨地監控數百台機器人的健康狀態與地圖位置。

常見問題解答

這裡有你想了解的技術細節與商務細則

支持哪些系統？

KairoIO 提供標準 REST API 與 SDK。目前原生支持 ROS 1, ROS 2, Arduino (ESP32), Python 與 Node.js 客戶端。

延遲表現如何？

在穩定的網際網路環境下，狀態數據動態同步延遲低於 50ms。視頻流通過 WebRTC P2P，通常在 100-200ms 內。

數據安全性？

所有數據傳輸均經過 SSL/TLS 加密。企業版支持完全私有化部署在客戶的內網或虛擬雲端 (VPC) 中。
