API 使用指南

所有受保護的 API 終端（Endpoints）都需要在 HTTP 標頭中包含 Authorization 並攜帶 Bearer Token。

Authorization: Bearer <access_token>

6.1 身份驗證

透過登入 API 取得 Access Token 與 Refresh Token。

curl -X POST http://localhost:8080/api/v1/auth/login \
  -H "Content-Type: application/json" \
  -d '{"username": "user@example.com", "password": "password"}'

6.2 機器人管理

將新的機器人註冊到車隊中。

curl -X POST http://localhost:8080/api/v1/robots \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "robot_id": "turtlebot-001",
    "robot_type": "turtlebot3",
    "robot_name": "倉庫機器人 #1"
  }'

6.3 狀態與遙測

定期更新機器人狀態（心跳、位置、電量）。

curl -X POST http://localhost:8080/api/v1/robots/{robotId}/status \
  -H "Authorization: Bearer $TOKEN" \
  -d '{"status": "active", "battery_level": 85}'

6.8 指令與控制

透過 REST 向機器人下發 Action 指令。

curl -X POST http://localhost:8080/api/v1/robots/{robotId}/control \
  -H "Authorization: Bearer $TOKEN" \
  -d '{"action": "gripper_open", "parameters": {"force": 10}}'
