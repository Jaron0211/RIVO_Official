API Usage Guide

All protected API endpoints require an Authorization header with a Bearer Token.

Authorization: Bearer <access_token>

6.1 Authentication

Obtain Access Token and Refresh Token via the login API.

curl -X POST http://localhost:8080/api/v1/auth/login \
  -H "Content-Type: application/json" \
  -d '{"username": "user@example.com", "password": "password"}'

6.2 Robot Management

Register a new robot to the fleet.

curl -X POST http://localhost:8080/api/v1/robots \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "robot_id": "turtlebot-001",
    "robot_type": "turtlebot3",
    "robot_name": "Warehouse Robot #1"
  }'

6.3 Status & Telemetry

Periodically update robot status (heartbeat, position, battery).

curl -X POST http://localhost:8080/api/v1/robots/{robotId}/status \
  -H "Authorization: Bearer $TOKEN" \
  -d '{"status": "active", "battery_level": 85}'

6.8 Command & Control

Send action commands to robots via REST.

curl -X POST http://localhost:8080/api/v1/robots/{robotId}/control \
  -H "Authorization: Bearer $TOKEN" \
  -d '{"action": "gripper_open", "parameters": {"force": 10}}'
