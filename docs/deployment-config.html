Deployment & Configuration

3. Deployment Options

3.1 Docker Deployment (Recommended)

Docker is the recommended way to deploy RIVO Server in production.

docker build -t rivo-server:latest .

Run with the required environment variables:

docker run -d \
  -p 8080:8080 \
  -e RIVO_AUTH_JWT_ACCESS_SECRET="your-32-char-secret" \
  -e RIVO_AUTH_JWT_REFRESH_SECRET="your-refresh-secret" \
  rivo-server:latest

3.2 Binary Deployment

Can run directly on Linux, macOS, or Windows. On Linux, systemd is recommended for daemon management.

4. Configuration Reference

RIVO can be configured via config.yaml file or environment variables (prefixed with RIVO_).

Core Configuration

Config Key Environment Variable Description
auth.jwt.access_secret RIVO_AUTH_JWT_ACCESS_SECRET JWT signing key (at least 32 characters)
database.type RIVO_DATABASE_TYPE "sqlite" or "postgres"
server.port RIVO_SERVER_PORT Default port: 8080
