Deployment & Configuration

1. RIVO Server — Docker Compose (Recommended)

The production deployment uses Docker Compose with two containers: rivo-server (API + LiveKit) and rivo-updater (OTA update agent).

Directory layout

deploy/docker/
├── docker-compose.yml
├── .env.example        # copy to .env and fill secrets
├── Dockerfile
└── Dockerfile.updater

.env setup

cp deploy/docker/.env.example deploy/docker/.env

Required variables in .env:

VariableExampleDescription
DB_PASSWORDchange_me_strongPostgreSQL password (also set in server config)
JWT_ACCESS_SECRET32+ random charsJWT access token signing key
JWT_REFRESH_SECRET32+ random charsJWT refresh token signing key

docker-compose.yml (key sections)

services:
  rivo-server:
    image: rivo-server:latest
    ports:
      - "8080:8080"   # REST API
      - "7880:7880"   # LiveKit WebSocket signaling
      - "7881:7881"   # LiveKit TCP ICE fallback
      - "7882:7882/udp" # LiveKit WebRTC media (UDP — CRITICAL)
    volumes:
      - /opt/rivo-server/rivo-server:/app/rivo-server  # OTA bind-mount
      - /var/run/docker.sock:/var/run/docker.sock
    env_file: .env
    restart: unless-stopped

  rivo-updater:
    image: rivo-updater:latest
    network_mode: "service:rivo-server"
    volumes:
      - /opt/rivo-server:/opt/rivo-server
      - /var/run/docker.sock:/var/run/docker.sock
    restart: unless-stopped
AWS / Cloud NAT: Ensure UDP port 7882 is open inbound in your security group. Without it WebRTC video will fail to establish.

Start

cd deploy/docker
docker compose up -d
docker compose logs -f rivo-server   # watch startup

Generated by make build. LiveKit keys are auto-generated randomly at build time — never hardcode them.

3. RIVO Node — Installation

Download binary

# ARM64 (Raspberry Pi 4/5, Jetson, Orange Pi, etc.)
wget https://github.com/Jaron0211/RIVO-Node_release/releases/latest/download/rivo-node-linux-arm64.zip
unzip rivo-node-linux-arm64.zip -d /opt/rivo-node
chmod +x /opt/rivo-node/rivo-node

Directory structure after install

/opt/rivo-node/
├── rivo-node              # main binary
├── configs/
│   └── config.yaml        # main configuration
├── protocols/             # sensor YAML files (add yours here)
├── interfaces/            # actuator JSON files (add yours here)
├── actuators/             # (legacy) actuator YAML files
└── safety_rules.yaml      # edge safety gate rules

systemd service

# /etc/systemd/system/rivo-node.service
[Unit]
Description=RIVO Node
After=network-online.target
Wants=network-online.target

[Service]
ExecStart=/opt/rivo-node/rivo-node --config /opt/rivo-node/configs/config.yaml
WorkingDirectory=/opt/rivo-node
Restart=on-failure
RestartSec=5s
User=root

[Install]
WantedBy=multi-user.target
sudo systemctl daemon-reload
sudo systemctl enable --now rivo-node
journalctl -u rivo-node -f   # follow logs

4. RIVO Node — config.yaml Full Reference

node

node:
  robot_id: ""            # leave empty to auto-derive from hardware MAC
  robot_type: "edge-node" # free text, shown in fleet list
  robot_name: "RIVO Node"
  group_id: ""            # optional fleet group UUID
  firmware_version: "1.3.0"  # reported in status, cached by server

server

server:
  url: "http://192.168.1.104:8080"  # RIVO Server URL
  timeout_seconds: 10
  telemetry_ws: false   # set true for persistent WebSocket telemetry
                        # (reduces HTTP overhead for high-frequency sensors)

auth

auth:
  mode: "access_token"    # "access_token" or "cert_key"
  username: "node@example.com"
  password: "your-password"
  cert_key: ""            # used when mode = cert_key
  access_token: ""        # auto-filled at runtime, do not set manually
  refresh_token: ""       # auto-filled at runtime

uarts — Multi-UART (named map)

uarts:
  uart0:
    enabled: true
    port: "/dev/ttyS0"
    baud: 9600
    framing: "fixed"      # "fixed" | "cobs"
    encoding: "binary"
    protocol: "uart-sen0177"  # must match id: in protocols/*.yaml
  uart1:
    enabled: false
    port: "/dev/ttyUSB0"
    baud: 115200
    framing: "fixed"
    encoding: "binary"
    protocol: "my-sensor"

i2c

i2c:
  enabled: false
  bus: 1        # /dev/i2c-1
  scan: true    # log detected addresses on startup

cameras

cameras:
  - name: "main"
    enabled: true
    type: "uvc"              # "uvc" or "csi"
    device: "/dev/video0"
    width: 1280
    height: 720
    framerate: 30
    gst_pipeline: ""         # optional: override entire GStreamer pipeline string

ui (LCD1602)

ui:
  enabled: true
  display: "lcd1602"
  i2c_bus: 1
  i2c_addr: "0x27"
  buttons: []
  leds: []

telemetry

telemetry:
  status_interval: "2s"      # how often to push robot status
  telemetry_interval: "500ms" # how often to push sensor telemetry

spool — Offline Buffering

spool:
  enabled: false
  dir: "/var/lib/rivo-node/spool"
  max_mb: 128        # disk cap for buffered telemetry when offline

protocols — IO File Locations

protocols:
  dir: "/opt/rivo-node/protocols"          # sensor YAML files
  interfaces_dir: "/opt/rivo-node/interfaces"  # actuator JSON files
  actuators_dir: "/opt/rivo-node/actuators"    # legacy actuator YAML

safety — Edge Safety Gate

safety:
  watchdog_interval: "5s"    # connectivity probe interval
  offline_threshold: 3        # consecutive failures before fallback triggers
  fallback_policy: "stop"     # "stop" | "hold" | "none"
                               # stop: zero all actuators; hold: keep last value
  rules_file: "/opt/rivo-node/safety_rules.yaml"

logging

logging:
  level: "info"   # "debug" | "info" | "warn" | "error"
