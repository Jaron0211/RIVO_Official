Getting Started

RIVO is an IoT robot fleet management platform consisting of three components that work together to connect physical hardware to a cloud management layer.

Note: This guide is intended for operators, DevOps engineers, and integration developers. It covers system architecture, deployment, and integration.

1. System Architecture

The RIVO platform has three components:

Component Technology Role
RIVO Server Go / PostgreSQL / LiveKit Cloud backend — robot registry, telemetry store, command dispatch, AI assistant, OTA update
RIVO Node Go binary (ARM64 / amd64) Edge daemon — reads hardware sensors, streams camera video, enforces safety rules, executes actuator commands
RIVO Android App Flutter Mobile client — real-time monitoring, live video, AI chat, remote command

2. Data Flow

  1. Sensor data — RIVO Node reads hardware (UART, I2C, GPIO) → pushes telemetry/status to Server via HTTP POST or persistent WebSocket.
  2. Video — Node captures camera via GStreamer (VP8/RTP) → publishes to LiveKit room. App subscribes to the same room and receives the video stream.
  3. Commands — App or AI issues commands via Server REST API → Server delivers to Node over WebSocket command channel → Node runs through Edge Safety Gate → executes actuator write.
  4. Realtime control — App connects to the same LiveKit room and sends DataChannel messages directly to Node for sub-20ms latency control.

3. Quick Start: RIVO Server

Step 1: Deploy with Docker Compose

The recommended production deployment uses Docker Compose. See Deployment & Config for the full docker-compose.yml.

cd deploy/docker
cp .env.example .env          # fill in DB_PASSWORD and JWT secrets
docker compose up -d

Step 2: Verify Health

curl http://localhost:8080/health
# {"status":"healthy","timestamp":"..."}

Step 3: Create Master Account

./rivo-admin seed \
  --email admin@yourcompany.com \
  --password "SecurePassword123!"

Step 4: Login (RSA-Encrypted)

All login credentials are RSA-encrypted before transmission. See API Usage Guide for the full authentication flow.

4. Quick Start: RIVO Node

Step 1: Install the Binary

# Download latest release
wget https://github.com/Jaron0211/RIVO-Node_release/releases/latest/download/rivo-node-linux-arm64.zip
unzip rivo-node-linux-arm64.zip -d /opt/rivo-node
chmod +x /opt/rivo-node/rivo-node

Step 2: Configure

Edit /opt/rivo-node/configs/config.yaml — at minimum set the server URL and credentials:

server:
  url: "http://<your-server-ip>:8080"

auth:
  mode: "access_token"
  username: "your-node-account@example.com"
  password: "your-password"

Step 3: Define Your Hardware

Place sensor protocol YAML files in /opt/rivo-node/protocols/ and actuator JSON files in /opt/rivo-node/interfaces/. See IO Definition for the format.

Step 4: Run

# Direct run
/opt/rivo-node/rivo-node --config /opt/rivo-node/configs/config.yaml

# Or as systemd service (recommended)
sudo systemctl enable rivo-node
sudo systemctl start rivo-node

On startup the Node will auto-register with the Server and begin streaming telemetry.

5. Platform Capabilities Summary

Capability Component Details
Declarative sensor protocols Node YAML files in protocols/ — no recompile needed
Declarative actuator interfaces Node JSON files in interfaces/ — auto-exposed to AI
Edge Safety Gate Node YAML rules block unsafe commands before execution
Offline watchdog Node Auto stops actuators on network failure
LiveKit video streaming Node + Server GStreamer → WebRTC VP8, low-latency
DataChannel realtime control Node + App P2P control via LiveKit DataChannel, <20ms latency
AI natural language control Server LLM-powered assistant with safety validation and actuator awareness
OTA update Server + Node Binary swap via rivo-updater agent, with rollback
E2E encryption Server + Node RSA login + AES-GCM on all protected payloads
Multi-UART / I2C / SPI / Modbus Node Simultaneous buses, named in config.yaml
