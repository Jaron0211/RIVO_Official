Security & Maintenance

9. Security Best Practices

  • JWT Keys: Ensure access_secret and refresh_secret are at least 32 characters long.
  • HTTPS: In production, use a reverse proxy (Nginx or Caddy) with TLS encryption.
  • Rate Limiting: Enable built-in rate limiting to prevent brute-force attacks.
  • Field Encryption: Use security.encryption_key to encrypt sensitive data in the database.

8. Operations & Maintenance

System Monitoring

Monitor system health via the /health endpoint.

Log Management

Logs are automatically rotated daily and stored in the logs/ folder. Set logging.level to debug for detailed traces.

Backup

For SQLite, directly backup the data/rivo.db file. For PostgreSQL, use pg_dump for regular backups.
