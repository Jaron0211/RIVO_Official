# RIVO 定價策略 PROMPT

## 產品定位
RIVO 是機器人產品孵化器服務（Robot Product Incubator Service），核心價值是幫助機器人團隊快速將原型變成可上線產品。

---

## 成本結構分析

### 主要成本來源
1. 雲端伺服器費用（CPU/RAM）- 考慮使用 AWS
2. 資料儲存空間（預設 200GB，可擴充）
3. 頻寬/流量（串流、資料傳輸）
4. 第三方服務費用（LiveKit、推播通知等）

### 成本影響指標

| 指標 | 影響程度 | 說明 |
|------|---------|------|
| 連線設備數量 | **低** | 不應作為主要定價依據 |
| 感測器/資料點數量 | **高** | 直接影響 API 呼叫和資料處理 |
| 控制頻率 | **高** | 1Hz vs 100Hz 差 100 倍負載 |
| 串流時數 | **高** | LiveKit 按量計費 |
| 資料保留天數 | **高** | 儲存成本線性增長 |
| API 呼叫次數 | **高** | 伺服器負載直接相關 |

### LiveKit 成本參考
- 頻寬：$0.12/GB 起
- 連線分鐘：$0.0005/分鐘（約 NT$0.5/小時）
- 錄影/導出：$0.02/分鐘（視訊）
- 可自建免費，但需要適當的伺服器資源

## 定價原則

### 核心理念
1. **以效能為主軸**：控制頻率、感測器數量比設備數量更能反映實際成本
2. **移除設備數量限制**：對客戶更有吸引力，且對成本影響低
3. **串流時數計費**：直接對應 LiveKit 成本
4. **資料保留分級**：儲存成本透明化

### 升級動機設計
```
Maker（免費）→ 控制頻率不夠、串流受限
    ↓
Lite / Academic → 專案變認真、需要更多感測器
    ↓
Pro → 需要更高控制頻率、更長串流
    ↓
Business → 企業需求、需要 SLA
    ↓
Enterprise → 完全客製化
```

## 定價策略備註

1. **不限制設備數量**：對客戶更有吸引力，且對我們成本影響低
2. **控制頻率是關鍵差異**：1Hz → 10Hz → 30Hz → 100Hz 明確的效能階梯
3. **串流時數直接對應成本**：LiveKit 按量計費，透明轉嫁
4. **教育方案獨立**：培養未來客戶，獲取案例
5. **孵化服務創造收入**：一次性費用補貼訂閱優惠
